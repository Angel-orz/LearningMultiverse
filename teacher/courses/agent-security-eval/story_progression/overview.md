# agent-security-eval 故事总览

> 场景：星穹列车停靠在了一个新的站点——前方是 Kaggle 大会的会场与安全与评估的战场。

## 情感阶段

| 阶段 | 课次 | 主题 | 故事氛围 |
|------|------|------|---------|
| 初期 — 边界 | Ch.1-2 | 安全边界在哪里 | 列车窗外飘着警示灯颜色的光。列车长帕姆给每个人都发了一张安全工卡。 |
| 中期 — 信任 | Ch.3-4 | 信任的坍塌与重建 | 当机器不听话的时候——谁是最后一道防线？三月翻了翻相机，里面有一张她还没给别人看过的照片。 |
| 后期 — 交付 | Ch.5-6 | 从安全到可用 | "它没做坏事"和"它做得好"是两个完全不同的标准。列车接近终点前的最后一段加速。 |

## 与之前课程的关系

本课程发生在 kaggle-agent 和 agent-tools-interoperability 课程之后。你已经在天文台学到了 Agent 工程化基础，在列车上学完了工具/协议/Skills——现在站在安全与评估的维度上，回头看之前学过的那些 Agent，思考它们如何被安全地约束、如何被可信地评估。

这是一门"回头看"的课。之前学的 MCP、A2A、Skills——它们都是工具。安全与评估是确保这些工具能被放心使用的框架。
