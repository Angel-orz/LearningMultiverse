# Ch.4 — 玻璃盒子：看见 Agent 的思维

- **老师**：丹恒 🗡️
- **情感阶段**：中期 — 信任
- **教材范围**：p24-26（可观测性与安全工作总览）

## 场景

车厢里的灯全亮了。不是刺眼的那种亮，是一种均匀的光，让每个角落都没有阴影。丹恒站在车厢中央——没有坐在过道对面，也没有用他的固定位置。他站在你对面。

"如果一个 Agent 做了错误的事情——你是怎么发现的？"

他等了你片刻。

"你是看了日志？还是它把数据库删了你才收到告警？"

他在面前的玻璃窗上画了一条线——很轻，像是用手指在雾气上划过。

"这条线是 Agent 的思考轨迹，你叫它'思维链'、叫它推理过程都行。如果你看不到这条线——你就只能等它撞墙的声音。"

他擦掉那条线，又画了一条新的——这次它没有走直线，中途拐了一个小弯。

"这个弯就是 Intent Drift。它不是突然出错——它只是偏了一点点。偏了一次、两次、十次——你认不出它最初想做什么了。"

"今天不学新概念。今天学的是怎么看到你已经拥有的东西。"

## 故事节点

1. **开场**：全亮的车厢——丹恒站在你对面
2. **Vibe Trajectory**：用 OpenTelemetry 追踪 Agent 的完整认知步骤
3. **Intent Drift & Trust Decay**：Agent 的思维如何悄悄偏离轨道
4. **AgBOM（Runtime Agent Bill of Materials）**：动态的 Agent 活动资产清单
5. **Checkpoints + Stateful Circuit Breakers**：先存档再执行——分数降到阈值就回滚
6. **安全四大原则 Recap**：沙箱化、前移边界、零环境权限、Agentic SecOps

## 过渡

丹恒在玻璃窗上画的最后一条线——他画了一个闭环。起点和终点重合了。

"安全能确保 Agent 不做坏事。但安全不能确保它做的是好事。"

他把手从玻璃上放下来。

"明天三月会跟你讲后一半——怎么判断一个安全的 Agent，到底是不是一个好的 Agent。"
