# Ch.2 — 沙箱之内：当代码活过来之后

- **老师**：丹恒 🗡️
- **情感阶段**：初期 — 边界
- **教材范围**：p13-17（沙箱与供应链防御）

## 场景

今天的车厢比平时暗一点——不是灯没开，是丹恒把窗帘拉下来了一半。过道对面的座位上放着一台终端——屏幕亮着，显示的是一个空的命令行提示符。

丹恒坐在终端旁边，保温杯在桌上，盖子没拧紧。

"你写了一段代码，让 Agent 去执行。"他的声音不高，没有开场白。"你敲了回车，然后这段代码就不在你的控制里了。它在哪跑？谁决定它能访问什么？"

他把终端往你这边推了一点。

"三月上次跟你说的是'笼子'。今天我们把笼子的结构拆开——沙箱、网络出口、依赖仓库。这三个东西决定了你的 Agent 到底是关在笼子里，还是以为自己关在笼子里。"

他停了一下，看了一眼窗外的光。

"最危险的笼子是那种你以为关好了——其实有一扇门没锁的。"

## 故事节点

1. **开场**：暗车厢 + 空终端——"你敲回车之前，代码已经不在你手里了"
2. **Ephemeral Sandboxing**：Vibe Loop 的本质决定每次执行都必须在隔离环境中
3. **Slopsquatting**：Agent 幻觉产生的包名被攻击者利用——自动下载的供应链攻击
4. **Egress Governance**：Non-Interactive Access + 离线缓存 + 代理
5. **应用层漏洞**：前端存储敏感操作、CI/CD 比 IDE 更可靠的强制策略、MCP Spoofing

## 过渡

丹恒在下课前把终端关了。屏幕暗下去的那一刻，他说了一句：

"明天三月的课，她会跟你讲身份的问题——当一个 Agent 有了名字和权限，它就不再只是一个程序了。"

他拧紧保温杯盖——这是他今天第一次碰到它。
