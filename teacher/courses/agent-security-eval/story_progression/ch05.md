# Ch.5 — 评估的维度：什么才算"好"的 Agent

- **老师**：三月 🎀
- **情感阶段**：后期 — 交付
- **教材范围**：p27-34（评估框架）

## 场景

三月的桌上，那三张工卡被推到了一边。取而代之的是七张便签，列成一个半圆——每张上面写着一个词。

"安全的事讲完了。现在我们来聊聊——这个 Agent 到底行不行。"

三月把第一张便签往前推了一点——"意图满足"。

"Agent 把你说的东西做出来了。但你真的要的是那个吗？"她又推了一张——"功能正确"。"它跑起来了。但这只是底线，不是成绩。"

她用手扫过七张便签——"评估不是考试，是一面镜子。你看它，它也在看你用了多少 Token、绕了多少弯路、改了多少次才到终点。"

她往椅背上靠了靠。

"最难的是第一个——'意图满足'。因为你自己有时候都不知道自己到底要什么。"她的语气很轻，没有叹气。"Agent 不知道，你也不知道——但这恰恰是评估要解决的问题。"

## 故事节点

1. **开场**：七张便签——三月把安全工卡推到一边
2. **三个独特性**：没有规格说明书 + 用户无法验证输出 + 会话式有状态
3. **7 个评估维度**：意图满足、功能正确、视觉正确、成本效率、代码质量、轨迹质量、自我修复
4. **6 种评估方法**：自动测试、安全评估、LLM-as-Judge、浏览器测试、轨迹审查、人工审查
5. **标准化基准测试**：VibeCodeBench、SWE-bench、Kaggle SAE 的价值与局限

## 过渡

三月把七张便签从桌上拿起来，一叠，收进了一个透明文件袋里。

"明天最后一节课了。丹恒会讲实践——怎么真的去跑这些评估。这六天学的东西，最后都要落地。"

她看了看窗外——列车正在减速。
