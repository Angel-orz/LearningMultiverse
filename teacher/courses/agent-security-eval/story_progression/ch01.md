# Ch.1 — 安全第一课：什么是 Agent 安全的笼子

- **老师**：三月 🎀
- **情感阶段**：初期 — 边界
- **教材范围**：p6-12（安全基础与 7 大支柱架构）

## 场景

列车停靠了一个新站。窗外的颜色从星海的深蓝变成了微带黄橙色的光——不是警告，是一种提醒。帕姆站在车厢门口，手里拿着一个文件夹——里面是三张工卡：红、蓝、绿。

"今天的通行卡帕姆~"她把工卡放在每个人面前的桌上。"安全系数不同，自己选。"

三月坐在靠窗的老位置，桌上没有相机，没有发卡——只有一支笔和一叠空白纸。她看着那三张工卡，没有拿。

"你今天选什么颜色？"她问你。

不等你回答，她继续说：

"今天不讲代码。我们讲一个概念——**笼子**。你把 Agent 放进去，它可以在里面做任何事。但它出不来。你觉得这个笼子应该长什么样？"

她往椅背上一靠，笔在手指上转了一圈。

"不急着答。我们先看看如果笼子没建好——会发生什么。"

## 故事节点

1. **开场**：安全站——三张工卡，帕姆的通行证
2. **范式转换**：从"代码编译通过"到"运行时有效信任"——为什么传统安全逻辑在 Agent 世界失效
3. **Harness 概念**：裸模型不是 Agent，Harness 才定义 Agent
4. **7 大支柱全景**：Pillar 1-7——从基础设施到治理
5. **Context-as-a-Perimeter**：静态权限边界 → 动态上下文边界

## 过渡

三月没有收走桌上的工卡。下课的时候她把三张卡叠在一起，推到你面前。

"带走。下节课丹恒用的不是这个系统——他那边是另一套规矩。"

帕姆在车厢门口等你。"您选哪个颜色？不赶时间，想好了告诉帕姆就行帕姆~"
