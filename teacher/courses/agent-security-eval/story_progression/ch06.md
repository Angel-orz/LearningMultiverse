# Ch.6 — 评估实践：落地、纠错、总结

- **老师**：丹恒 🗡️
- **情感阶段**：后期 — 交付
- **教材范围**：p35-41（评估实践与总结）

## 场景

列车在减速。窗外的星海正在淡出——取而代之的是越来越近的地面灯光。终点站快到了。

丹恒今天坐在暖气和窗之间那个不常坐的位置——不是他的固定座位，但他选了那里。保温杯放在桌上，盖子是拧紧的。

"上周我们学了安全——笼子、身份、可观测。三月跟你讲了评估的七个维度。今天是最后一节课——怎么真的把这些东西跑起来。"

他没有用白板。他直接打开了一个终端。

"第一件事：没有可观测性就没有评估。你看不到它就测不了它。所以在你评估之前——先确定你能不能看到它的脑海里在想什么。"

他敲了几行命令——不是真的要运行什么，更像是演示节奏。

"第二件事：把用户的前两条消息当作评估标准。Vibe Coding 没有需求文档——用户自己都不知道。但他说的第一句话，就是他最真实的需求。"

他关掉终端，转过来看着你。

"第三件事：用户的每一次纠正，都是免费的标注数据。'\这里不对\''\我要的不是这个\'——收集、聚类、分析。这是一个不断累积的失败模式库。"

他停了一下。

"最后一件事——不要看代码，看产物。用户不会打开文件来评价你的工作，他们看到的是渲染出来的页面。"

窗外，终点站的灯光越来越亮。丹恒没有说结束语。他只是在保温杯的盖子上轻轻按了按——已经拧紧了，他只是确认了一遍。

## 故事节点

1. **开场**：减速的列车——丹恒坐在不常坐的位置
2. **Observability 是评估的前提**：OpenTelemetry 追踪的三大 Span 类型
3. **Tail-Based Dynamic Sampling**：如何在生产环境中控制追踪成本
4. **Session Prefix as Intent Rubric**：以用户的起始消息派生出接收标准
5. **评估渲染产物而非代码**：多模态模型直接评分截图 vs 代码分析
6. **Session Convergence 模型**：关注整段会话收敛而非单轮准确率
7. **用户纠错聚类**：Labeled Failure Data → Systematic Gap Analysis

## 收束

车厢广播响了："前方到站——终点站。请整理好您的随身物品。"

丹恒站起来，拿走保温杯。他在车厢门口停了一下——没有回头。

"这六课从头到尾走了一遍——从安全到评估，从笼子到镜子。你学的不是一个框架——是一种判断力。"

他没有等你回答，走出了车厢。

过了一小会儿，三月从另一节车厢探出头来。"诶——你还在车上啊？到站了。"她笑了一下，那台胶片相机挂在她的脖子上。"需要帮忙搬一下笔记吗？"
