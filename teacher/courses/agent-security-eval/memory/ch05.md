## 关键推理链条

```
安全：Agent 没做坏事
评估：Agent 做的事值得发布
  ↓
三个独特性：
  1. Underspecification Gap（没有规格说明）
  2. 用户无法验证输出
  3. 会话迭代 + 代码库状态
  ↓
7 个评估维度：
  Intent Satisfaction（最难）→ Functional Correctness（底线）
  → Visual Correctness → Cost & Efficiency → Code Quality
  → Trajectory Quality → Self-Repair
  ↓
6 种评估方法：
  Auto Test → Security Eval → LLM-as-Judge
  → Browser Test → Trajectory Inspection → Human Review
  ↓
标准化基准：VibeCodeBench / SWE-bench / Kaggle SAE
  ↓
陷阱：Benchmark Overfitting → 真实世界意图不匹配
```
