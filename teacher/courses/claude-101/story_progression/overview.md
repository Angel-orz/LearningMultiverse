# 故事总览 — Claude 101

> 星穹列车的新旅程：从贝洛伯格出发，驶向「知识之域」——一个由信息本身构成的世界。
> 这次，开拓者要学习的不再是 ML 或 Agent 工程，而是如何与列车上的新伙伴——Claude——这个 AI 助手真正协作。

---

## 核心设定

本次旅程的背景设定在**知识之域**（The Domain of Knowledge），一个由信息流和数据构成的抽象世界。列车驶入这片领域后，三月和丹恒发现——这里的"居民"是各种信息实体，而唯一能和它们有效沟通的方式，是借助 Claude 的能力。

开拓者的任务：学会使用 Claude，帮助知识之域的居民解决他们的信息困境。

---

## 章节概览

| 课次 | 情感阶段 | 故事背景 | 核心教学主题 |
|------|---------|---------|------------|
| Ch.1 | 好奇与探索 | 列车驶入知识之域，遇见第一个信息实体——困惑的档案管理员 | 什么是 Claude？它和普通聊天机器人有什么不同 |
| Ch.2 | 初试锋芒 | 档案管理员请求帮助整理混乱的文献，开拓者尝试第一次真正的对话 | 如何有效对话、提问题、让 Claude 理解你的意图 |
| Ch.3 | 遇到瓶颈 | 一个复杂的信息实体提出了模糊的需求，简单的对话已经不够了 | 迭代追问、提供上下文、调整输出格式 |
| Ch.4 | 工具觉醒 | 知识之域的"项目大厅"——需要把零散信息组织成结构 | 使用 Projects 组织工作 |
| Ch.5 | 创意爆发 | 一位艺术家信息实体需要帮助——不仅仅是文字，还要创造视觉内容 | Artifacts + Skills |
| Ch.6 | 走出列车 | 知识之域出现了外部入侵——来自其他系统的信息乱流 | 工具连接 + Enterprise Search |
| Ch.7 | 终极考验 | 知识之域的中央图书馆需要一次全面的信息救援 | 综合应用 + 结业 |

## 关键主题（暗线）

- **从聊天到协作**：Claude 不是搜索引擎，而是思考伙伴
- **提示词即编程**：好的提问方式决定了答案的质量
- **知识管理**：Projects、Artifacts、Skills 是 Claude 世界的组织工具
- **边界意识**：知道 Claude 能做什么，也知道它不能做什么
